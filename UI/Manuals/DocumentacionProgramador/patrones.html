Patrones Utilizados

Composite (Permisos)

Permite modelar jerarquías de permisos mediante Patentes y Familias. Una Familia contiene otras Familias o Patentes. La SL interpreta la estructura para determinar acceso a funcionalidades.

Repository

Aísla el acceso a datos del resto del sistema. Permite reemplazar SQL por archivos sin modificar la lógica del negocio.

Factory

Define qué implementación de repositorio se utilizará según configuración (SQL o File).

Service Layer

Patrón de capa de servicios que centraliza reglas transversales: Logs, seguridad, DVH/DVV, validaciones y orquestación.
