Arquitectura de SIGACF

La arquitectura del sistema sigue un modelo en capas claro y desacoplado. Cada capa cumple una responsabilidad específica para garantizar separación de responsabilidades, reutilización y claridad.

Domain Layer

Contiene las entidades del negocio (Cliente, Reserva, Pago, Usuario, Promoción, etc.). No contiene lógica, solo estado y estructura.

DAL (Data Access Layer)

Encargada del acceso a datos. Implementa repositorios con soporte para archivos y SQL Server. Se utiliza el patrón Factory para seleccionar dinámicamente el backend.

BLL (Business Logic Layer)

Implementa las reglas del negocio, validaciones de entidades y excepciones personalizadas. No conoce detalles de la UI ni de la persistencia.

SL (Service Layer)

Es la capa central del sistema. Orquesta las operaciones, registra logs, controla permisos, aplica DVH/DVV, maneja seguridad y sirve como interfaz entre la UI y el resto del sistema.
